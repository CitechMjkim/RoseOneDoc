    ❗️ RefreshView 안에는 CommonLayoutView를 사용하여 작업합니다. ( componets/CommonLayoutView.tsx )

    #example

    <CommonLayoutView
        data={TIDALHOMESECTION}
        extraData={queries}
        renderItem={({item, index}) => (
        )}
    />

    • 이 단계에서 데이터는 useQueries를 사용하여 호출합니다. ( 예시: TidalHome.tsx 38 ~ 82)

    • extraData는 FlashList에서 UI가 렌더링 된 후 데이터가 호출될 시 리스트를 리렌더링 하는 기능을 합니다.

    • TIDALHOMESECTION 변수는 화면의 섹션 구성 데이터를 정의한 배열입니다. 각 항목은 하나의 콘텐츠 섹션을 나타내며, 섹션의 제목, 콘텐츠 타입, API 호출 함수, 파라미터, 리스트 형태 등을 포합합니다.

      해당 변수는 ISectionDataInfo 타입을 기반으로 정의됩니다.

    export const TIDALHOMESECTION = [
        {  title: TEXT_CONFIG.str_music_mix, type: 'playlist', param: [""], function: "getTidalMix", all: false, listType: "default" },
        {  title: TEXT_CONFIG.str_video_mix, type: 'playlist', param: [""], function: "getTidalMix", all: false, listType: "default" },
        {  title: TEXT_CONFIG.str_suggest_new, type: 'album', param: ["new"], function: "getTidalFeaturedPath", all: true, listType: "default" },
        {  title: TEXT_CONFIG.str_popular_album, type: 'album', param: ["top"], function: "getTidalFeaturedPath", all: true, listType: "default" },
        {  title: TEXT_CONFIG.str_master_playlist, type: 'playlist', param: [""], function: "getTidalMaster", all: true, listType: "default" },
        {  title: TEXT_CONFIG.str_master_album, type: 'album', param: [""], function: "getTidalMaster", all: true, listType: "default" },
        {  title: TEXT_CONFIG.str_focus, type: 'playlist', param: ["concentrate"], function: "getTidalMood", all: false, listType: "default" },
        {  title: TEXT_CONFIG.str_tidal_rising, type: 'playlist', param: [""], function: "getTidalRising", all: true, listType: "default" },
      ] as ISectionDataInfo[]

      {
        title: 제목
        type: album, playlist...
        param: API 호출 시 사용한 파라미터 배열,
        function: API 호출 시 사용한 함수명,
        all: 모두보기 유무,
        listType: 리스트 타입 ( 자세한 내용은 Storybook에서 확인 )
      }
    • title은 TEXT_CONFIG에 정의해서 사용합니다.
    • 변수명은 str_ 로 시작합니다

    ❗️ CommonLayoutView안에서는 각각의 Section을 순차적으로 렌더링합니다. Section에서는 CommonSectionView를 사용하며 SectionHeader와 DefaultRowList 를 사용하여 제목과 리스트를 나타냅니다.

    example

    const getData = (name:string) => {
        switch(name){
            case TEXT_CONFIG.str_music_mix :
                return mixTrack?.data || []
            case TEXT_CONFIG.str_video_mix :
                return mixVideo?.data || []
            case TEXT_CONFIG.str_suggest_new :
                return suggestNew?.data?.items || []
            case TEXT_CONFIG.str_popular_album :
                return popularAlbum?.data?.items || []
            case TEXT_CONFIG.str_master_playlist :
                return masterPlaylist?.data?.items || []
            case TEXT_CONFIG.str_master_album :
                return masterAlbum?.data?.items || []
            case TEXT_CONFIG.str_focus :
                return mood?.data?.items || []
            case TEXT_CONFIG.str_tidal_rising :
                return rising?.data?.items || []
            default:
                return []
        }
    }

    <CommonSectionView>
        <SectionHeader>
            <CommonText type="title" text={item.title}/>
        </SectionHeader>
        <DefaultRowList data={getData(item.title)} entityType={MENU_ENTITY.TIDAL} type={item.type} listType={item.listType} isLoading={queries[index]?.isLoading} />
    </CommonSectionView>
    • getData 함수를 이용해 현재 섹션에 아이템을 렌더링 한다.

    React-query


    queryKey: [MENU_ENTITY, Route, "MixTrack", token],
    //MENU_ENTITY - ROSE_HOME, TIDAL, APPLE_MUSIC...
    //Route - Home, Explore, Setting
    • queryKey에 순서는 현재 페이지 정보 [MENU_ENTITY, Route] - 페이지 refresh에 사용됩니다. 유니크 키 [MixTrack] - 같은 페이지에서 캐시 키로 활용됩니다. 유저 토큰 [token] - 유저 정보가 변경되었을때 리렌더링으로 활용됩니다.
            // 캐시 지우고 새로운 데이터 불러오기기
            queryClient.invalidateQueries({ 
                queryKey: [MENU_ENTITY, Route] 
            });
    • queryKey에 순서가 같아야 전체 새로고침이 작동됩니다.

    Style

    • Color, font, font-size는 contsants/Color.ts에 정의된 값만 사용합니다.
    • Color에 Type 정의는 type/theme/Theme.ts에서 합니다.

    const example = styled.Text`
         background-color: ${({ theme }) => theme.colors.text};
         font-size: ${({ theme }) => theme.font.md};
         font-family: ${({ theme }) => theme.font.bold}
    `

    const exampleProp = styled.Text<{ isFocus: boolean}>`
         background-color: ${({ theme }) => isFocus ? theme.color.mainColor  : theme.colors.text};
         font-size: ${({ theme }) => theme.font.md};
         font-family: ${({ theme }) => theme.font.bold}
    `
    • style에서 prop을 전달해야하는 경우 exampleProp 형태로 사용합니다.
